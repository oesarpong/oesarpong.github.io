    Get Time Readings Command

    Expanding on the previous task, this command, when called, sends the current states of both the data arrays for time and temperature. Both of these arrays are continuously being updated to display the latest data points. The data is received by my laptop, which stores the new string characteristic and processes it accordingly for time and temperature (see Notification Handler).

    The effective transfer rate using this method is also t = 25.0 ms. The temperature readings are reported in degrees Fahrenheit.

    Remarks

    To my surprise, these two methods present different transfer rates. Collecting data in the Arduino IDE program creates consistent intervals, which can be internally set by the user. Therefore, the "quickness" of the data collection is determined by the delay. Conversely, collecting the data individually with the notification handler is dependent on the speed of your computer.

    I believe that both methods present advantages and disadvantages. If the user is looking for consistent data intervals, then the second method is favorable. However, as these labs continue, it is likely that multiple commands will be found in the Artemis' main loop, which could accumulate a large delay. Using the notification handler will interrupt the script, potentially saving the user time in the end.

    Hypotheically if the Artemis has 384 kB of RAM then we can store approximately 96,000 data points at 4 bytes per. These points are to be divided between all the data arrays in this case those for Time and Temperature (this is excluding the memory required for the script and variables).
